Event-Driven TCP Server.
Это мой проект, на основе которого я хочу научиться взаимодействию с сетью, на практике понять и развить:
- как работает tcp;
- многопоточность и событийное дерево;
- навыки системного програмирования;
- развить навык описания архитектуры приложений;
- улучшу навыки управление памятью 

Пока архитектура планируется вот такая:
```
          Clients
             │
             ▼
      ┌──────────────┐
      │ TCP Listener │   accept()
      └──────┬───────┘
             ▼
   ┌───────────────────┐
   │ Event Loop (epoll)│  ← один поток
   └──────┬────────────┘
          │ events (read/write)
          ▼
   ┌───────────────────┐
   │ Connection Object │
   └──────┬────────────┘
          │ task
          ▼
   ┌───────────────────┐
   │   Thread Pool     │  ← N worker threads
   └───────────────────┘
```
Основные компоненты:
1. прослушиватель - проверяет наличие новых событий
2. отправитель задачи в pool
3. Connection (соединение) 
4. дерево с pool

Как это будет работать:
1. подключается клиент
2. ему присваивается fd
3. fd добавляеться в epoll
4. выполняется запрос пользователя
5. пользователю отдается ответ
6. соединение либо отключаеться, либо остается закрытым
