Что планируется реализовать:
- **Завершить и формализовать state machine соединения**
- **Развести серверную логику и обработку клиентов по потокам**
- **Реализовать полноценную запись (write-side)**
- **Улучшить обработку ошибок и аварийных ситуаций**

---

## Что уже реализовано

- **Non-blocking TCP сервер**
  - серверный и клиентские сокеты работают в non-blocking режиме;
  - используется системный вызов `epoll`.

- **Event-driven архитектура**
  - один event loop обрабатывает все сетевые события;
  - сервер масштабируется по количеству соединений, а не потоков.

- **Протокол поверх TCP**
  - сообщения передаются в формате `[length][payload]`;
  - корректно обрабатываются склеенные и частично полученные пакеты.

- **Stateful чтение данных**
  - реализована логика `READ_LEN / READ_BODY`;
  - поддерживается partial read;
  - соединение корректно реагирует на `EAGAIN / EWOULDBLOCK`.

- **Базовая модель соединения**
  - каждому клиенту соответствует объект `Connection`;
  - хранится состояние соединения и буферы данных.

- **Клиент для тестирования**
  - реализован TCP-клиент;
  - поддерживается корректная отправка сообщений с partial write;
  - используется `writev`.
